## 二、三大通用框架模板

### 框架1: 教程类内容框架 (最常用)

**适用场景**: AI工具教程、技巧分享、操作指南

**完整结构**:

```
第1幕: 开头Hook (10-20秒)
├─ 痛点场景描述: "你是不是也遇到..."
└─ 价值承诺: "今天教你..."

第2幕: 快速建立信任 (10-20秒)
├─ 自我证明: "我用这个方法..."
└─ 适用人群声明: "这套方法适合..."

第3幕: 核心教学 (60-120秒)
├─ 工具介绍 (如需要)
├─ 步骤1: 准备工作
├─ 步骤2: 核心操作演示
├─ 步骤3: 避坑提示/高级技巧
└─ 效果展示/对比

第4幕: 行动号召 (10-20秒)
├─ 一句话总结
└─ 互动引导(收藏/评论/关注)
```

**时长分配** (以3分钟视频为例):
- 开头: 15秒
- 建立信任: 15秒
- 核心教学: 2分钟
- 结尾: 30秒

**关键要点**:
- ✓ 前30秒必须抓住注意力
- ✓ 演示要清晰可复制
- ✓ 每个步骤都要有字幕标注
- ✓ 结尾必须有行动指令

**案例参考**: ChatGPT教程、Midjourney使用技巧

---

### 框架2: 测评对比框架

**适用场景**: 工具测评、产品对比、功能评测

**完整结构**:

```
第1幕: 测评背景 (10-20秒)
├─ 测评动机: "最近很多人问..."
└─ 测评对象: "我实测了A和B"

第2幕: 测评维度说明 (10秒)
└─ 列出3-5个对比维度

第3幕: 逐项对比展示 (90-120秒)
├─ 维度1: 功能对比 + 演示
├─ 维度2: 效果对比 + 截图
├─ 维度3: 价格对比
└─ 维度4: 使用体验

第4幕: 结论与建议 (20-30秒)
├─ 总体评分
├─ 推荐场景
└─ 选择建议
```

**时长分配** (以5分钟视频为例):
- 开头: 30秒
- 维度说明: 10秒
- 对比展示: 3分30秒
- 结论: 50秒

**关键要点**:
- ✓ 对比要公平客观
- ✓ 数据要真实可验证
- ✓ 结论要明确不模棱两可
- ✓ 提供具体选择建议

**案例参考**: ChatGPT vs Claude、免费AI工具对比

---

### 框架3: 经验分享框架

**适用场景**: 学习经验、方法论、心路历程

**完整结构**:

```
第1幕: 个人背景与成果 (15-30秒)
├─ 起点描述: "一年前我是..."
└─ 成果展示: "现在我已经..."

第2幕: 核心观点提出 (10-20秒)
└─ 核心方法论/洞察

第3幕: 方法论展开 (90-150秒)
├─ 方法1: 具体做法 + 案例
├─ 方法2: 具体做法 + 案例
├─ 方法3: 具体做法 + 案例
└─ (可选)方法4-5

第4幕: 行动建议 (20-30秒)
├─ 总结回顾
└─ 鼓励尝试
```

**时长分配** (以4分钟视频为例):
- 背景成果: 30秒
- 核心观点: 20秒
- 方法展开: 2分40秒
- 行动建议: 30秒

**关键要点**:
- ✓ 成果要可信可量化
- ✓ 方法要具体可操作
- ✓ 避免纯理论,要有实例
- ✓ 激励而非炫耀

**案例参考**: AI学习方法、产品经理成长经验

## 三、不同内容类型的框架差异

### 3.1 图文 vs 视频差异

| 维度 | 图文内容 | 视频内容 |
|------|---------|---------|
| 开头 | 1-2张图快速吸引 | 前3秒必须有冲击力画面 |
| 节奏 | 可以跳读,信息密度高 | 需要控制语速,留呼吸感 |
| 演示 | 多用截图标注 | 录屏+画外音+字幕 |
| 结尾 | 文字行动指令 | 口播+字幕双重强调 |

### 3.2 短视频 vs 长视频差异

| 维度 | 短视频(1-3分钟) | 长视频(5-15分钟) |
|------|----------------|-----------------|
| 开头 | 5秒内必须Hook | 可以有30秒铺垫 |
| 内容 | 只讲1个核心点 | 可以讲3-5个要点 |
| 深度 | 点到为止 | 可以深入展开 |
| 节奏 | 快速紧凑 | 可以有缓急变化 |

### 3.3 小红书 vs 抖音差异

| 维度 | 小红书 | 抖音/视频号 |
|------|--------|------------|
| 首图/封面 | 图文并茂,信息丰富 | 人物特写/冲击画面 |
| 标题 | 可以更长,更详细 | 短而有力,直击痛点 |
| 内容密度 | 可以较高,用户愿意看 | 需要降低,快速过完 |
| 互动方式 | 评论区深度交流 | 点赞/转发为主 |

## 四、时长分配黄金比例

### 4.1 按视频长度分配

**1分钟视频**:
- 开头Hook: 10秒 (17%)
- 核心内容: 45秒 (75%)
- 结尾CTA: 5秒 (8%)

**3分钟视频**:
- 开头Hook: 20秒 (11%)
- 问题引入: 20秒 (11%)
- 核心内容: 2分钟 (67%)
- 结尾总结: 20秒 (11%)

**5分钟视频**:
- 开头Hook: 30秒 (10%)
- 背景铺垫: 30秒 (10%)
- 核心内容: 3分30秒 (70%)
- 结尾行动: 30秒 (10%)

### 4.2 按内容模块分配

**开头Hook (必须精彩)**:
- 占比: 10-15%
- 目标: 3秒内抓住注意力
- 方法: 痛点/数据/反差/提问

**核心内容 (价值主体)**:
- 占比: 60-70%
- 目标: 提供实际价值
- 方法: 演示/案例/步骤

**结尾CTA (行动指令)**:
- 占比: 5-10%
- 目标: 引导互动
- 方法: 提问/福利/预告

## 五、框架选择决策树

```
开始
│
├─ 是否有明确操作步骤?
│  ├─ 是 → 使用"教程类框架"
│  └─ 否 ↓
│
├─ 是否需要对比多个选项?
│  ├─ 是 → 使用"测评对比框架"
│  └─ 否 ↓
│
├─ 是否分享经验方法论?
│  ├─ 是 → 使用"经验分享框架"
│  └─ 否 ↓
│
└─ 混合使用,根据内容调整
```

## 六、框架优化检查清单

在内容发布前,用以下清单检验框架:

**结构完整性**:
- [ ] 是否有明确的开头Hook?
- [ ] 是否在前30秒内建立了信任?
- [ ] 是否提供了核心价值/解决方案?
- [ ] 是否有清晰的行动指令?

**节奏把控**:
- [ ] 开头是否够快够吸引?
- [ ] 中间是否有节奏变化?
- [ ] 是否避免了冗长铺垫?
- [ ] 结尾是否简洁有力?

**用户体验**:
- [ ] 用户能否快速get到价值?
- [ ] 内容是否易于理解和复制?
- [ ] 是否解决了用户的实际问题?
- [ ] 是否给了用户明确的下一步?

**平台适配**:
- [ ] 时长是否符合平台特性?
- [ ] 开头是否符合推荐算法?
- [ ] 互动引导是否符合平台习惯?
