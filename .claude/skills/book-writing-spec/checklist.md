# 交稿前自查清单

> 全部通过后方可提交。标注 ⭐ 的为高频错误项，重点核查。

---

## 一、基础设置

- [ ] 正文行距设置为 1.15
- [ ] 使用 Word 2010/2013/2016 版本（非 macOS）
- [ ] 所有编号均为手动输入（未使用 Word 自动编号功能）

---

## 二、标题格式

- [ ] 一级标题：三号黑体，居中
- [ ] 二级标题：小三号黑体，顶格
- [ ] 三级标题：四号黑体，顶格
- [ ] 四级标题：小四号黑体，前空两格，词组形式，句末无标点
- [ ] 五级标题：五号黑体，词组形式，句末无标点
- [ ] 标题中无括号、破折号等注释（已移至正文）
- [ ] 目录只提取前三级，不显示页码

---

## 三、正文

- [ ] ⭐ 文中英文单词无双引号（如 `Next` 而非 `"Next"`）
- [ ] 每段开头空两个汉字位置
- [ ] 无明显口语化表述（"我们""咱们""大家""来聊一聊"等）
- [ ] 科技数字使用阿拉伯数字
- [ ] "注意"格式正确：`注意：`（黑体） + 正文（楷体）

---

## 四、插图

- [ ] ⭐ 图题在图的**下方**（不是上方）
- [ ] 图按章编号，格式为 `图X-X`
- [ ] 正文中有引用，格式为 `如图X-X所示`
- [ ] 图的文字描述在上方，图在下方
- [ ] ⭐ 图中英文已翻译为中文（或提供了对照表）
- [ ] 截图分辨率 ≥ 1000 dpi
- [ ] 并排图片已用 (a)(b)… 标注

---

## 五、插表

- [ ] ⭐ 表题在表的**上方**（不是下方）
- [ ] 表按章单独编号，格式为 `表X-X`（不与图混编）
- [ ] 正文中有引用，格式为 `见表X-X`
- [ ] 表格各项最后一句无标点

---

## 六、代码

- [ ] ⭐ 代码使用了**代码清单**样式
- [ ] 代码用 `代码如下：` 引出，命令用 `命令如下：` 引出
- [ ] ⭐ 多于5行的代码，开头有注释写明文件路径（`//第X章/xxx.xxx`）
- [ ] 代码注释使用中文，句末无标点

---

## 七、公式

- [ ] 公式按章编号，格式为 `(X-X)`
- [ ] 正文引用格式为 `在式(X-X)中`
- [ ] 单字母变量用斜体，多字母变量/常量用正体
- [ ] 矢量/张量/矩阵用黑斜体

---

## 八、交稿要求

- [ ] 全书页数在 350～550 页（最好 300～600 页之间）
- [ ] 所有内容合并为一个 Word 文件（含内容简介、前言、目录、正文、参考文献）
- [ ] 同时导出了 PDF 文件
- [ ] 官方链接未加下划线
- [ ] 个人链接已与编辑沟通处理方案

---

## 九、函数与方法

- [ ] ⭐ 函数和方法名后有括号（`add()` 而非 `add`）
