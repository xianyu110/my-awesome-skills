# 字流多平台内容分发技能

## 核心理念

> **一次创作，多端分发；让内容创作者专注于创作本身，把繁琐的平台适配交给AI。**

这个技能帮助内容创作者实现：
- 从一篇长文自动生成14个平台的适配内容
- 自动处理不同平台的字段差异、格式要求、内容规则
- 生成平台专属的标题、描述、标签、封面、口播稿
- 最大化内容的分发效率和影响力

## 支持的平台矩阵

### 长文平台（4个）
1. **微信公众号** - 允许二维码、微信引导
2. **知乎** - 需要更学术化的表达
3. **掘金** - 技术向，代码友好
4. **知识星球** - 支持批量发布多个星球

### 短图文平台（5个）
5. **小绿书** - 年轻化表达，emoji友好
6. **小红书图文** - 需要精美封面+标签
7. **微博** - 话题标签+@互动
8. **即刻** - 简洁直接，社区调性
9. **Twitter** - 英文适配，hashtag优化

### 视频平台（5个）
10. **视频号** - 微信生态，中老年友好
11. **抖音** - 年轻化，热点导向
12. **小红书视频** - 精致生活化
13. **B站** - 二次元文化，弹幕互动
14. **YouTube** - 国际化，SEO优化

## 工作流程

### Step 1: 内容源识别

**输入格式支持**：
- Markdown文件
- 飞书文档导出
- 纯文本
- HTML（微信公众号导出）

**操作**：
1. 读取原始内容
2. 识别内容类型（技术/产品/生活/教程等）
3. 提取核心信息：
   - 主题
   - 关键观点
   - 数据/案例
   - 行动号召（CTA）


### Step 2: 平台规则分析

**不同平台的核心差异**：

#### 内容规则差异
- **公众号**：可以放二维码、微信号、外链
- **知乎/掘金**：不能直接引导加微信，需要更隐晦
- **小红书**：不能有明显营销，要生活化
- **抖音**：不能有联系方式，要用评论区引导
- **Twitter**：可以放链接，但要简洁

#### 格式支持差异
- **Markdown支持**：掘金、知乎（部分）
- **富文本**：公众号、知识星球
- **纯文本+图片**：小红书、微博、即刻
- **视频+文案**：视频号、抖音、B站

#### 字段要求差异
- **标题长度**：公众号64字 / 小红书20字 / 抖音30字
- **描述长度**：B站2000字 / 抖音100字
- **标签数量**：小红书10个 / B站12个 / 抖音30个
- **封面尺寸**：各平台不同（16:9 / 3:4 / 1:1）

### Step 3: 内容改写策略

**核心原则**：保持核心信息不变，调整表达方式和结构

#### 长文平台改写
```
原文 → 平台适配版本

公众号版：
- 保持完整内容
- 开头：个人化引入 + 痛点共鸣
- 结尾：二维码 + 微信引导 + 往期推荐

知乎版：
- 保持完整内容
- 开头：问题导向 + 数据支撑
- 结尾：关注引导（不能放微信）+ 相关回答推荐

掘金版：
- 技术细节增强
- 代码块优化
- 开头：技术背景 + 解决方案
- 结尾：GitHub链接 + 技术交流

知识星球版：
- 增加独家内容/思考
- 社群互动引导
- 提问/讨论话题
```

## 核心原则

### ✅ 必须遵守

1. **内容一致性**：核心信息不变，只调整表达方式
2. **平台合规**：严格遵守各平台规则
3. **真实性**：不夸大，不编造数据
4. **用户视角**：保持原作者的语气和观点

### ❌ 绝不允许

1. **违规内容**：不生成违反平台规则的内容
2. **过度改写**：不改变原文核心意思
3. **标题党**：不使用虚假夸张的标题
4. **抄袭**：不使用他人的原创表达
