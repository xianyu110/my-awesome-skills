# 三遍审校清单

## 第一遍：内容审校（逻辑、事实、结构）

### 事实准确性
- [ ] 所有数据都有来源（真实测试/官方信息/权威报道）
- [ ] 产品名称、版本号、时间等信息准确
- [ ] 技术细节正确，无错误描述
- [ ] 引用的案例真实，无编造
- [ ] 数字和统计数据准确

### 逻辑清晰性
- [ ] 论点明确，论据充分
- [ ] 前后无矛盾
- [ ] 因果关系合理
- [ ] 推理过程清晰
- [ ] 结论有支撑

### 结构合理性
- [ ] 开头引入自然
- [ ] 各部分衔接流畅
- [ ] 重点突出，无跑题
- [ ] 结尾有总结或升华
- [ ] 整体结构完整

### 无编造内容
- [ ] 所有案例都是真实的
- [ ] 所有数据都是真实的
- [ ] 所有观点都有依据
- [ ] 没有为了效果而夸大

## 第二遍：风格审校（AI味降重）

### 删除套话

常见套话清单（全部删除）：
- [ ] "在当今时代"
- [ ] "在...的背景下"
- [ ] "综上所述"
- [ ] "总而言之"
- [ ] "值得注意的是"
- [ ] "不难发现"
- [ ] "可以说"
- [ ] "众所周知"
- [ ] "毋庸置疑"
- [ ] "显而易见"

### 拆解AI句式

常见AI句式（需要拆解）：
- [ ] "不是...而是..."（连续出现）
- [ ] "既...又..."（连续出现）
- [ ] "一方面...另一方面..."（连续出现）
- [ ] "不仅...而且..."（连续出现）
- [ ] "虽然...但是..."（连续出现）

**改写方法**：
- 拆成多个短句
- 用更自然的表达
- 避免对仗句式

### 替换书面词汇

常见书面词汇（改成口语化）：

| 书面词汇 | 口语化表达 |
|---------|-----------|
| 显著提升 | 具体数字（快了多少） |
| 充分利用 | 用好 |
| 进行操作 | 具体动词（点击、输入等） |
| 实现了 | 做到了 |
| 获得了 | 拿到了 |
| 达到了 | 到了 |
| 完成了 | 做完了 |
| 开展了 | 做了 |
| 推进了 | 推了 |
| 落地了 | 上线了 |

### 改成口语化

检查清单：
- [ ] 多用短句（15-25字）
- [ ] 多用动词，少用名词化表达
- [ ] 多用主动语态，少用被动语态
- [ ] 多用具体词汇，少用抽象词汇
- [ ] 多用"你"、"我"，少用"用户"、"开发者"

**改写示例**：

❌ "通过对工具的深入使用，开发者可以获得更好的开发体验。"
✅ "你多用几次，就知道这工具有多好用了。"

❌ "该功能的实现为用户提供了更多的可能性。"
✅ "有了这个功能，你能做的事更多了。"

### 加入真实细节

检查清单：
- [ ] 抽象表达 → 具体数字
- [ ] 泛泛而谈 → 具体案例
- [ ] 中立客观 → 明确观点
- [ ] 理论分析 → 实践经验

**改写示例**：

❌ "这个工具能显著提升开发效率。"
✅ "用这个工具，我平均每个项目省3-5小时。"

❌ "很多开发者都在使用这个工具。"
✅ "我问了10个朋友，8个都在用。"

### 加入个人态度

检查清单：
- [ ] 有明确的观点（不是中立客观）
- [ ] 有个人的判断（不是罗列事实）
- [ ] 有真实的感受（不是理性分析）
- [ ] 有态度的表达（不是平铺直叙）

**改写示例**：

❌ "这个工具有优点也有缺点。"
✅ "这个工具确实好用，但有个问题让我很烦。"

❌ "用户可以根据自己的需求选择。"
✅ "如果你是开发者，我建议直接用这个。"

## 第三遍：细节打磨（标点、排版、节奏）

### 句子长度

检查清单：
- [ ] 大部分句子15-25字
- [ ] 没有超过30字的长句
- [ ] 长短句结合，有节奏变化

**改写方法**：
- 找出超过30字的长句
- 拆成2-3个短句
- 用句号，不用逗号连接

**改写示例**：

❌ "Claude Code是一个基于AI的编程工具，它不仅可以生成代码，还可以理解项目结构，给出更合理的建议，从而帮助开发者提升开发效率。"（52字）

✅ "Claude Code是一个AI编程工具。它不只是生成代码。它能理解你的项目结构，给出更合理的建议。用好它，开发速度能快不少。"（4句，平均15字）

### 段落长度

检查清单：
- [ ] 手机屏幕3-5行（约80-150字）
- [ ] 没有超过200字的长段落
- [ ] 段落之间有空行

**改写方法**：
- 找出超过200字的长段落
- 拆成2-3个段落
- 每个段落一个中心意思

### 标点使用

检查清单：
- [ ] 多用句号，少用逗号连接长句
- [ ] 适当使用问号（引发思考）
- [ ] 适当使用感叹号（表达态度，但不过度）
- [ ] 避免使用分号、冒号等复杂标点

### 节奏变化

检查清单：
- [ ] 快慢结合（短句 + 长句）
- [ ] 疏密结合（密集信息 + 留白）
- [ ] 张弛结合（紧张 + 放松）

**技巧**：
- 重要信息：用短句，快节奏
- 解释说明：用长句，慢节奏
- 转折过渡：用中等长度句子

### 大声朗读

检查清单：
- [ ] 读起来顺畅，不拗口
- [ ] 节奏自然，不生硬
- [ ] 语气合适，不做作

**方法**：
- 大声朗读全文
- 找出读起来不顺的地方
- 调整句式和用词

## 常见问题

### Q: 如何判断是否有AI味？
A: 看这几点：
- 有没有套话（"在当今时代"等）
- 有没有AI句式（"不是...而是..."连续出现）
- 有没有书面词汇（"显著提升"等）
- 有没有真实细节（具体数字、具体案例）
- 有没有个人态度（明确观点）

### Q: 如何快速降低AI味？
A: 三个技巧：
1. 删除套话
2. 加入具体数字和案例
3. 改成口语化表达

### Q: 如何保持用户的风格？
A: 参考用户的历史文章：
- 开头方式
- 结构偏好
- 语言特征
- 金句风格

### Q: 如何避免过度修改？
A: 记住核心原则：
- 保持原有观点和态度
- 保持原有结构和逻辑
- 只改表达方式，不改核心内容
